## Structure des updates sur le serveur d'update ##

-current.off
 Si le fichier existe et contient la date et l'heure à laquelle le serveur sera Online, alors le serveur d'update est à OFF.
 Si le fichier n'existe pas ou il est vide, alors le serveur est à ON.

-current.version
 Numéro de la version de l'update la plus récente

-client2/# (Où # signifie le numéro de l'update) (Optionel)
	- /files.list (1 entrée par ligne séparé par §)
	  Copie le fichier du serveur de MAJ vers le client (dossier du programme Clinica) et crée le dossier nouveau s'il n'existe pas
	  Il doit donc avoir un fichier uploadé sur le serveur d'update par ligne
	  0-Emplacement (<- ou)et nom du fichier (Ex: NewFile.bmp OU NewDossier/NewFile.bmp)
	  1-Emplacement (<- ou)et nom du fichier (Ex: NewFile.bmp OU NewDossier\NewFile.bmp)
	  2-Taille du fichier
	  3-Fichier nécessitant l'updater (True|False) (ces fichiers sont habituellement déjà en cours d'utilisation, ex: Clinica.exe)
	  4-Mode de téléchargement (Binary|ASCII)
	
	- /major.update
	  Fichier "FLAG" indiquant que cette mise à jour nécessite un redémarrage

-server2/# (Où # signifie le numéro de l'update) (Optionel)
	- /files.list (1 entrée par ligne séparé par §)
	  Copie le fichier du fichier vers le serveur (dossier des données) et crée le dossier nouveau s'il n'existe pas
	  Il doit donc avoir un fichier uploadé sur le serveur d'update par ligne
	  0-Emplacement (<- ou)et nom du fichier (Ex: NewFile.bmp OU NewDossier/NewFile.bmp)
	  1-Emplacement (<- ou)et nom du fichier (Ex: NewFile.bmp OU NewDossier\NewFile.bmp)
	  2-Taille du fichier
	  3-Fichier nécessitant l'updater (True|False) (actuellement, toujours Faux)
	  4-Mode de téléchargement (Binary|ASCII)

	- /sqlupdates.sql   (Inclut dans files.list)
	  Fichier contenant les mises à jour SQL à effectuer sur la base de données Clinica.
	  Il peut y avoir plusieurs batch d'exécution séparés par des GO précédés et suivis de Enter.
	
	- /major.update
	  Fichier "FLAG" indiquant que cette mise à jour nécessite un redémarrage

	- /Updates/*.exe (Inclut dans files.list)
	  Programme qui sera exécuté au prochain lancement de Clinica et sera supprimé immédiatement après.

			- SQLStatement
			  Commande T-SQL pour afficher le rapport. Peut inclure la variable WHEREGEN qui sera remplacée par GeneralWhereStr & SQLWhere combinés.
			  La commande doit être du format :
			  SELECT Colonne1 AS TitreColonne1, Colonne2, FUNCTION(Colonne3)
			  FROM From
			  GROUP BY Colonne1,Colonne2
			  ORDER BY Colonne1,Colonne2
			  Voici l'explication des variables ds la commande :
				- Colonne1 : Nom de la colonne (peut être précéder du nom de la table et d'un point) qui contient les données de la première colonne des statistiques (Ex : noms des thérapeutes)
					     Les résultats peuvent apparaître de multiple fois, mais seront réduit à une seule ligne.
				- TitreColonne1 : Titre de la colonne #1.
				- Colonne2 : Nom de la colonne (peut être précéder du nom de la table et d'un point) qui contient les données des titres de toutes les autres colonnes des statistiques (Ex : noms des codes de dossier)
					     Les résultats ici ne peuvent pas avoir de doublons, car les nombres de la troisième colonne seront distribués sous chaque colonne
				- FUNCTION : Une function d'agrégat SQL quelconque. (Ex : Count, Sum)
				- Colonne3 : Nom de la colonne (peut être précéder du nom de la table et d'un point) qui sera utilisé pour créer les chiffres des statistiques
					     Les résultats de cette colonne apparaîtront sur la ligne de la Colonne1 et dans la colonne de la Colonne2.
				- From : Tout combinaison de table valide pour obtenir les statistiques souhaitées.
			  En gros, BodyStats nécessite une table à trois colonnes dont les deux colonnes du bout seront inversés (colonne/ligne) et dont la première sera éliminée les doublons

			  Peut inclure les variables sous la forme ###VARIABLE### qui sont ajoutés via le code.

			  Peut inclure les variables sous la forme ###VARIABLE### qui sont ajoutés via le code.
			- ExtraColumnsSQLWhere
			  Partie 'WHERE' d'une commande TSQL pour ExtraColumnsSQLWhere. Ce 'WHERE' est spécifique à tous les rapports créés par ce type.
			- TotalPosition
			  Position du total, soit en colonne ou en rangée (COL|ROW|BOTH)
			- ShowInPourcent
			  Affiche les données en pourcentage sur le total soit de la colonne (Choisir TotalPosition=ROW) ou de la rangée (Choisir TotalPosition=COL).
			  (Vrai|Faux)
			- TotalsType
			  Type d'action définit les totaux 
			  Peut prendre les valeurs suivantes : (SUM|AVG)
				SUM = Somme des éléments (Par défaut)
				AVG = Somme des éléments diviser par le nombre d'éléments (non nul)
		- BodyVisites
			- (Les Propriétés de BodyTable sont incluses)
			- NbColumnAfterVisites
			  Nombre de colonnes à afficher après le tableau de statut des visites par jour
			- FolderColName
			  Nom de la colonne dans SQLStatement qui représente le numéro de dossier de la visite
			- SQLVisitesStatuts
			  Commande T-SQL pour obtenir le statut des visites demandées. Peut inclure la variable WHEREGEN qui sera remplacée par GeneralWhereStr & SQLWhere combinés.
			  La commande doit être du format :
			  SELECT Colonne1, Colonne2, Colonne3
			  FROM From
			  Voici l'explication des variables dans la commande :
				- Colonne1 : Nom de la colonne qui contient la date de la visite
				- Colonne2 : Nom de la colonne qui contient le statut de la visite
				- Colonne3 : Nom de la colonne qui contient le numéro de dossier de la visite
				- From : Nom de la table duquel les informations seront prises.
	9- RapportFooterName
	   Nom de la classe pour ce pied de rapport (Le nom réel de classe est précédé de Rapport) (BasicFooter|FooterTotal|FooterSimple)
	10- RapportFooterProperties
	   Propriétés du corps du rapport (De la forme §NOMPROPRIÉTÉ§=§VALEUR§)
	   Noms des propriétés par classe:
		- BasicFooter
			- StyleFileName
			  Chemin partiel ou complet d'un fichier CSS. Peut inclure la variable ###CLINICAPATH###.
			- GeneralWhereStr
			  Propriété chargée dynamiquement via les filtres du rapport
			- ColsFormat
			  Format spécifique pour certaines colonnes.
			  {COLNAME=FORMAT£COLNAME=FORMAT£...}
			  COLNAME signifie le nom de la colonne à formatter
			  FORMAT peut prendre les valeurs suivantes : (INT|DATE|DATETIME|TIME|CURRENCY|CURRENCYNAME|STRING|POURCENT)
			  Lorsque le format String est spécifié, un comptage du nombre d'occurence sera affiché dans les sous-totaux.
		- FooterTotal
			- (Les Propriétés de BasicFooter sont incluses)
			- FooterFileName
			  Chemin partiel ou complet d'un fichier HTML. Peut inclure la variable ###CLINICAPATH###.
			  Le fichier HTML doit inclure les variables ###SUMTABLEFOOTER### et ###SUMOFLINES###.
		- FooterSimple
			- (Les Propriétés de BasicFooter sont incluses)
			- SQLStatement
			  Commande T-SQL pour afficher le rapport. Peut inclure la variable WHEREGEN qui sera remplacée par GeneralWhereStr & SQLWhere combinés.
			- SQLWhere
			  Partie 'WHERE' d'une commande TSQL pour SQLStatement. Ce 'WHERE' est spécifique à tous les rapports créés par ce type.
			- FooterFileName
			  Chemin partiel ou complet d'un fichier HTML. Peut inclure la variable ###CLINICAPATH###.
			  Le fichier HTML doit inclure les variables ###NOMDUCHAMPS###  et la variable ###ExtraField### (Définit par le filtre passif) à l'endroit où l'information doit apparaître.

			  Façon dont le champs en extra sera rempli (NONE|FILTER|ASK)
